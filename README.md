# 📧 Spam Email Detection using Naive Bayes

## 📌 Project Overview

This project implements a **Spam Email Detection System** using three different Naive Bayes algorithms:

* Gaussian Naive Bayes
* Multinomial Naive Bayes
* Bernoulli Naive Bayes

The objective is to classify SMS messages as **Spam** or **Ham (Not Spam)** and compare the performance of all three models.

---

# 🎯 Objectives

* Detect spam SMS messages using Machine Learning.
* Convert text messages into numerical features.
* Train and compare three Naive Bayes algorithms.
* Evaluate models using Accuracy, Classification Report, and Confusion Matrix.
* Identify the best-performing Naive Bayes model for text classification.

---

# 📂 Dataset

**Dataset Name:** SMS Spam Collection Dataset

**Features**

| Column  | Description |
| ------- | ----------- |
| label   | Spam or Ham |
| message | SMS message |

The dataset contains **5,572 SMS messages** labeled as either **Spam** or **Ham**.

# ⚙️ Workflow

1. Load the dataset.
2. Select the required columns.
3. Rename columns.
4. Convert labels into numeric values.
5. Split features and target.
6. Convert text into numerical vectors using CountVectorizer.
7. Split the dataset into training and testing sets.
8. Train Gaussian, Multinomial, and Bernoulli Naive Bayes models.
9. Predict spam and ham messages.
10. Compare model performance using evaluation metrics.

---

# 🧠 Machine Learning Algorithms

## 1. Gaussian Naive Bayes

* Best for continuous numerical data.
* Requires dense input (`toarray()`).

---

## 2. Multinomial Naive Bayes

* Best for text classification.
* Uses word frequency counts.
* Works directly with sparse matrices.

---

## 3. Bernoulli Naive Bayes

* Best for binary features.
* Uses word presence or absence.

# 📊 Results

| Model                   | Accuracy | Best Use Case                         |
| ----------------------- | -------- | ------------------------------------- |
| Gaussian Naive Bayes    | ~89%     | Continuous numerical features         |
| Multinomial Naive Bayes | ~99%     | Text classification using word counts |
| Bernoulli Naive Bayes   | ~98%     | Binary text features                  |

---

# 💡 Conclusion

Among the three algorithms, **Multinomial Naive Bayes** achieved the highest accuracy because it is specifically designed for word-count features generated by **CountVectorizer**. Bernoulli Naive Bayes also performed well by considering the presence or absence of words, while Gaussian Naive Bayes was less suitable because text data does not follow a Gaussian distribution.
